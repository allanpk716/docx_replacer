# Excel 数据支持用户指南

## 概述

DocuFiller 支持使用 Excel (.xlsx) 文件作为数据源，可以保留上标、下标等格式信息。系统支持两种 Excel 列格式，并能自动检测当前文件使用的格式。

## Excel 文件格式

系统支持两种列格式，程序会根据文件内容自动判断使用哪种格式，无需手动配置。

### 格式自动检测

程序读取第一个非空行的 A 列（第一列）内容进行判断：

- **如果 A 列内容匹配 `#关键词#` 格式** → 两列模式（A 列为关键词，B 列为值）
- **如果 A 列内容不匹配 `#关键词#` 格式** → 三列模式（A 列为 ID，B 列为关键词，C 列为值）

> **提示**：确保数据从第一行开始，不使用表头行。程序会逐行读取所有非空行。

### 两列格式（关键词 | 值）

| A 列（关键词） | B 列（值）      |
|----------------|-----------------|
| #产品名称#     | D-二聚体        |
| #规格#         | 2x10⁹           |

- **A 列**：关键词，必须以 `#` 开头和 `#` 结尾
- **B 列**：对应的值，支持富文本格式（上标、下标等）

### 三列格式（ID | 关键词 | 值）

| A 列（ID） | B 列（关键词） | C 列（值）        |
|------------|----------------|-------------------|
| 001        | #产品名称#     | D-二聚体          |
| 002        | #规格#         | 2x10⁹             |
| 003        | #批号#         | BN20240101        |

- **A 列**：行标识符（ID），用于标识每一行数据，不可重复
- **B 列**：关键词，必须以 `#` 开头和 `#` 结尾
- **C 列**：对应的值，支持富文本格式（上标、下标等）

> **注意**：在三列格式中，A 列的内容不能是 `#xxx#` 形式，否则会被误判为两列格式。建议使用数字编号或纯文本作为 ID。

### 创建格式化内容

1. 在 Excel 中输入值
2. 选中要格式化的部分（如 "9"）
3. 右键 → 设置单元格格式 → 字体 → 上标/下标

## 使用流程

### 1. 准备 Excel 数据文件

按照上述格式创建 Excel 文件，输入关键词和对应的值。根据需要选择两列或三列格式。

### 2. 选择 Word 模板

在 DocuFiller 中选择包含内容控件的 Word 模板。

### 3. 选择 Excel 数据文件

点击"浏览"按钮，选择准备好的 Excel 文件。

### 4. 预览数据

系统会自动验证并显示数据预览。如有错误，会显示详细信息。

### 5. 开始处理

点击"开始处理"按钮，生成带格式的 Word 文档。

## JSON 转 Excel

如果您有旧的 JSON 数据文件，可以使用内置的转换工具：

1. 点击主窗口的"JSON转Excel转换工具"链接
2. 选择要转换的 JSON 文件
3. 选择输出目录
4. 点击"开始转换"

转换后，您可以在 Excel 中添加需要的格式。

## 验证规则

系统会严格验证 Excel 文件，不同格式下的验证规则有所不同：

### 通用验证规则

- ✅ 文件扩展名必须是 `.xlsx`
- ✅ 关键词必须以 `#` 开头和 `#` 结尾
- ✅ 关键词不能重复
- ⚠️ 值列为空会产生警告

### 三列格式额外验证

- ✅ A 列（ID）不能重复 — 如果存在重复 ID，验证将报错并列出重复的 ID
- ⚠️ A 列（ID）为空的行会被跳过，不参与 ID 重复检查

### 两列与三列验证差异

| 验证项       | 两列格式         | 三列格式             |
|--------------|------------------|----------------------|
| 关键词格式   | ✅ 检查          | ✅ 检查              |
| 关键词重复   | ✅ 检查          | ✅ 检查              |
| 值列为空     | ⚠️ 警告          | ⚠️ 警告              |
| ID 重复检查  | — 不适用         | ✅ 检查（报错）      |

## 常见问题

### Q: 转换后的文档没有显示格式？

A: 确保在 Excel 中正确设置了格式（上标/下标），然后在 Word 中检查内容控件的样式设置。

### Q: 可以同时使用 JSON 和 Excel 吗？

A: 可以，程序同时支持两种格式。但建议逐步迁移到 Excel 以获得更好的格式支持。

### Q: 转换工具可以批量处理吗？

A: 是的，转换工具支持多选文件批量转换。

### Q: 如何选择两列和三列格式？

A: 程序会自动检测。如果第一行的 A 列内容是 `#关键词#` 形式，则识别为两列格式；否则识别为三列格式。

### Q: 三列格式的 ID 列有什么用？

A: ID 列用于标识每一行数据，方便在 Excel 中管理和查阅。程序会检查 ID 是否重复以避免数据错误。
